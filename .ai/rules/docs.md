# Rules: Docs, Specs, And Shared Markdown Contracts

> Load this file when modifying repo-level markdown that informs implementation.

---

## Scope

This includes:

- `.ai/` governance files
- product or architecture specs
- shared task contracts
- glossary and prompt guidance

## Rules

1. Keep docs aligned with the current repo, not just the target vision.
2. Mark planned structure as planned when it does not exist yet.
3. Update docs when public contracts, shared terminology, or routing rules
   change.
4. Keep wording specific enough that future agents can act on it.

## Common Mistakes

- Describing future directories as if they already exist
- Leaving the glossary stale after introducing new business language
